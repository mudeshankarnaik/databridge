1. Project Intent & Scope

DataBridge Pro is a specialized middleware platform designed to automate the extraction of data from Hadoop/Hive ecosystems into ClickHouse for sub-second analytical querying. It centralizes credential management, SQL versioning, and automated job scheduling into a single governed interface.

2. System Architecture

The system is built on a decoupled architecture to separate the Management Layer from the Execution Engine.

Web Application (app.py)

A Streamlit-based interface that handles session-based authentication, interactive SQL authoring, and manual ad-hoc pushes.

Automation Worker (worker.py)

A headless background service that continuously monitors bridge_metadata.db to trigger scheduled sync tasks based on time-based logic.

3. Operational Process Flow

The sequential journey of a synchronization task:

LOGIN ➔ CREDENTIALS ➔ WRITE SQL ➔ AD-HOC PUSH ➔ SCHEDULE

4. User Interface Map

The portal is divided into five logical tabs. Access is restricted based on the user's assigned role.

Profile Tab

Mandatory setup for Hive and ClickHouse connections. Use Test Connection to validate network routes before saving.

Transformer Tab
  • Query Editor: Multi-line SQL workspace.
  • Preview & Schema: Executes a LIMIT 10 on Hive to auto-generate the ClickHouse table structure.
  • Library: Sidebar containing Approved Library (Global) and Local Workspace (User-specific).
Deploy & Schedule

Configuration of automation. Defines Sync Nature (Replace vs Append) and Frequency (Daily/Monthly/Hourly/Minutes).

Dashboard

Real-time monitoring of task statuses (Success/Failure) and row counts for auditing.

5. Data Transformation Logic

Column Sanitization Logic

To prevent ClickHouse ingestion errors, the portal applies the following regex-based cleaning to all incoming Hive columns:

re.sub(r'[^a-zA-Z0-9_]', '_', col_name)
Dynamic Type Mapping Engine

8. Debugging Protocols

Credential Issues
  • Verify Username and Password in the Profile tab. Check for case sensitivity.
  • Ensure the user account exists in the users metadata table.
  • Still facing issues? Connect with the System Admin for credential reset or account unlock.
SQL Pushing Issues
  • Validate Codes: Test the HiveQL logic in a separate command-line tool first.
  • Check Aliases: Ensure AS names do not contain special characters or spaces.
  • Verify that the target ClickHouse table isn't locked by another process.
Connectivity Issues
  • Check Hive: Ensure Host/Port 10000 is reachable from the server.
  • Check ClickHouse: Ensure Port 8123 (HTTP) is open and reachable.
  • Check network routing: Ensure the DataBridge Pro server is on the correct VPN/subnet to talk to the data lake.
